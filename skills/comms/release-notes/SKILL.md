## Writing Rules Applied
Generate the release notes following these principles:

1. **Lead with customer benefit** — "Now you can X" not "We added feature X"
2. **Write for the user reading at 8 PM on mobile** — short sentences, clear value
3. **Avoid engineering language** — no "refactored," "migrated," "modularized"
4. **Every feature answers:** what changed, who it helps, how to use it
5. **Bug fixes are customer-facing:** "Fixed issue where [user action] caused [bad outcome]"

## Tone Adaptation
Adjust tone based on audience:
- **Developer-facing:** more technical, appreciate implementation details
- **Business user:** focus on time saved, cost reduced, risk mitigated
- **Consumer:** casual, benefit-first, emoji optional
- **Enterprise:** formal, focus on compliance, reliability, admin controls
